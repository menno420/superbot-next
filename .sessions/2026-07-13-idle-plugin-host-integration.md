## The idle-side blocker (fixed upstream first)

idle's plugin manifest declared `capabilities=("idle",)`. This host's namespace
validator requires a 3-part `a.b.c` capability facet
(`sb/namespace/validate.py` `_CAPABILITY_PARTS=3`; `sb/spec/authority.py:160-161`
`capability_not_3_part`), so the joint compile reds a bare single-token
capability:

```
joint compile FORMAT_ERROR (namespace) capability:idle: capability_not_3_part
```

idle's own CI is sb-free (`importorskip("sb.spec.manifest")`), so its compiler
never ran and the blocker went unnoticed. Fixed upstream in `superbot-idle`
PR #85 (`capabilities=("idle.game.play",)` — matching this host's
`<subsystem>.game.<action>` games convention: `blackjack.game.play`,
`casino.game.play`, `counting.game.play`, `chain.game.play`). **This host PR
stacks conceptually on that fix and vendors the FIXED copy.** The exact
capability string is a namespace choice open to owner redirect (flagged on both
PRs).
